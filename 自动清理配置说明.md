# 🗑️ 录音文件自动清理配置说明

## 📋 功能说明

**自动删除 24 小时前的录音文件，避免存储压力**

- ✅ 每天自动执行
- ✅ 清理 24 小时前的录音
- ✅ 同时删除文件和数据库记录
- ✅ 显示清理统计信息
- ✅ 释放服务器存储空间

---

## 🔧 配置方法（2 选 1）

### 方法 1：宝塔面板计划任务（推荐）

#### 1️⃣ 上传清理脚本

```bash
# 上传文件到服务器
上传 clean_old_recordings.php 到：
/www/wwwroot/luyin.fzzyd.com/api/clean_old_recordings.php
```

#### 2️⃣ 添加计划任务

**宝塔面板 → 计划任务 → 添加任务**

**配置如下：**

| 配置项 | 值 |
|--------|-----|
| 任务类型 | Shell 脚本 |
| 执行周期 | 每天 |
| 小时 | 3 |
| 分钟 | 0 |
| 脚本内容 | `php /www/wwwroot/luyin.fzzyd.com/api/clean_old_recordings.php` |

**说明：**
- 每天凌晨 3:00 执行
- 此时服务器负载最低
- 不影响白天正常使用

#### 3️⃣ 保存并测试

**点击"保存"后，点击"执行"按钮测试：**


## 📊 清理策略说明

### 清理规则

| 录音时间 | 是否清理 | 说明 |
|---------|---------|------|
| 24 小时内 | ❌ 不清理 | 用户可能需要导出 |
| 超过 24 小时 | ✅ 清理 | 已超出导出时间窗口 |

### 清理内容

1. **录音文件** - 从服务器删除
2. **数据库记录** - 从 recordings 表删除
3. **统计信息** - 显示释放空间大小

### 为什么是 24 小时？

- ✅ 用户有一天时间导出录音
- ✅ 避免存储压力过大
- ✅ 平衡用户体验和存储成本
- ✅ 可根据需求调整（修改脚本中的 `-24 hours`）

## ⚙️ 调整清理时间

**如果需要调整清理时间窗口（如改为 48 小时）：**

**编辑 `clean_old_recordings.php`，找到第 24 行：**

```php
// 原代码（24 小时）
$cutoff_time = date('Y-m-d H:i:s', strtotime('-24 hours'));

// 改为 48 小时
$cutoff_time = date('Y-m-d H:i:s', strtotime('-48 hours'));

// 改为 72 小时
$cutoff_time = date('Y-m-d H:i:s', strtotime('-72 hours'));
```
